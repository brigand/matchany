# Filter (condition)

Behaves the same as `And` but with the argument order swapped. This allows you to add
filters on conditions but retain the left value.

```js
import match from 'pattahn';
import Filter from 'pattahn/cond/Filter';
import Prop from 'pattahn/cond/Prop';
import Id from 'pattahn/cond/Id';
```

It can be used for a simple match.

```js
const VALUE = { foo: 1 };

const result = Filter(Id(), Prop('foo', 1)).exec(VALUE);

expect(result).toEqual([VALUE]);
```

And is also exposed as a method on any `Condition`.

```js
const VALUE = { foo: 1 };

const result = match()
  .with(Id().filter(Prop('foo', 1)), (x) => 'match: ' + x.foo)
  .exec(VALUE);

expect(result).toBe('match: 1');
```
